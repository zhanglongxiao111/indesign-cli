# AI 协作文档

本目录用于存放与 AI 协作相关的过程材料。这里记录任务、报告、外部问答和用户反馈，不替代正式技术决策。

## 分类说明

| 分类 | 放置内容 | 流转方式 |
| ---- | -------- | -------- |
| `本地Agent/` | 可在本地执行的 Agent 任务、审查报告、排查结果 | 先放 `进行中/`，完成后整个主题目录移到 `归档/` |
| `外部咨询/` | 需要人工复制粘贴给外部 AI 或专家的提问、回复、追问 | 先放 `进行中/`，完成后整个主题目录移到 `归档/` |
| `用户反馈/缺陷/` | 用户发现的功能问题、复现步骤、截图和修复跟踪 | 按日期建文件，解决后可移入 `归档/` |
| `用户反馈/改进/` | 体验优化、流程优化、能力增强建议 | 按日期建文件，采纳或关闭后可移入 `归档/` |
| `用户反馈/计划/` | 由反馈沉淀出的功能计划或阶段计划 | 按日期建文件，完成后可移入 `归档/` |

## 命名约定

- 主题目录：`YYYY-MM-DD_主题`
- 本地 Agent 任务：`任务_主题_Agent.md`
- 本地 Agent 报告：`报告_主题_Agent.md`
- 外部咨询提问：`提问_主题.md`
- 外部咨询回复：`回复_主题.md`
- 用户反馈：`YYYY-MM-DD_简短描述.md`

## 生命周期

1. 创建任务或反馈时，先放入对应的 `进行中/` 或分类目录。
2. 文档开头写清背景、目标、当前结论、相关文件路径和约束。
3. 结论被正式采纳后，沉淀到 `docs/技术决策/` 或更新对应当前文档。
4. 已完成的主题目录及时移入 `归档/`，避免 `进行中/` 长期堆积。
