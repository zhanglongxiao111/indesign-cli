# AI 协作规范

本规范只定义 `docs/AI协作/` 下的文档怎么放、怎么命名、怎么流转。

## 1. 适用范围

- 本地 Agent 自动化任务和报告
- 外部 AI 或专家咨询往返材料
- 用户反馈的缺陷、改进和计划

## 2. 分类规则

| 类型 | 目录 | 说明 |
| ---- | ---- | ---- |
| 本地 Agent | `docs/AI协作/本地Agent/进行中/` | 本地可执行的任务、扫描、审查和报告 |
| 外部咨询 | `docs/AI协作/外部咨询/进行中/` | 需要人工复制粘贴给外部系统的材料 |
| 用户缺陷 | `docs/AI协作/用户反馈/缺陷/` | 功能问题、复现步骤、实际结果、期望结果 |
| 用户改进 | `docs/AI协作/用户反馈/改进/` | 体验优化、交互调整、流程优化建议 |
| 用户计划 | `docs/AI协作/用户反馈/计划/` | 从反馈沉淀出的阶段计划或功能计划 |

## 3. 命名规则

主题目录使用：

```text
YYYY-MM-DD_主题
```

本地 Agent 文件使用：

```text
任务_主题_Agent.md
报告_主题_Agent.md
```

外部咨询文件使用：

```text
提问_主题.md
回复_主题.md
追问_主题.md
回复_追问_主题.md
```

用户反馈文件使用：

```text
YYYY-MM-DD_简短描述.md
```

## 4. 文档开头必须交代

- 背景
- 目标
- 当前结论
- 相关文件路径
- 当前约束或待确认事项

## 5. 流转规则

- 本地 Agent 和外部咨询先建在 `进行中/YYYY-MM-DD_主题/`。
- 完成后将整个主题目录移入同类 `归档/`。
- 用户反馈按 `缺陷/`、`改进/`、`计划/` 分类长期保存。
- 反馈关闭后，可以移入对应分类的 `归档/`。
- 截图、样例文件和附件放在同级或同分类的 `assets/`。

## 6. 与其他文档区的关系

- 临时协作和证据：`docs/AI协作/`
- 正式技术决策：`docs/技术决策/`
- 架构和调用链路：`docs/系统地图/`
- 复杂缺陷复盘：`docs/bugfix/`

如果协作结论已经变成长期约束，不要只留在本目录，必须同步到正式文档。
