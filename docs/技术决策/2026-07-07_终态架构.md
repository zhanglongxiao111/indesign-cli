# 终态架构长期决策

日期：2026-07-07

状态：已落地，长期遵守。

## 背景

终态架构重构已把旧工具定义、handler 和 MCP switch 结构物理删除。当前仓库不再通过 `src/handlers/`、`src/types/` 或 `src/core/InDesignMCPServer.js` 维护工具能力。

本决策把已落地的长期约束沉淀到 `docs/技术决策/`，避免后续文档或实现回到旧结构。

## 决策

### 1. tool-module 共置

`src/tools/index.js` 是当前工具真相 registry。

每个内置工具的事实必须共置在 `src/tools/<domain>/` tool-module 内：

- MCP tool name
- JSON Schema
- contract
- handler
- CLI id / alias

新增或修改工具的标准动作：

1. 修改对应 `src/tools/<domain>/` tool-module。
2. 由域 `index.js` 聚合；新域再接入全局 `src/tools/index.js`。
3. 执行 `node src/core/artifact.js --write`。
4. 执行 `node src/core/artifact.js --check`。
5. 更新架构基线和测试。

不得新增第二套工具映射、手写 help 目录、Python schema 复制或旧 handler facade。

### 2. profiles / internal 模型

MCP 暴露由 registry profile 决定：

| registry profile | MCP 暴露 | artifact / CLI source |
| ---------------- | -------- | --------------------- |
| `classic` | 经典 MCP server 暴露 | `classic` |
| `advanced` | 高级模板 MCP server 暴露 | `advanced` |
| internal | MCP 不暴露 | `hidden_handler` |

当前代码中 internal 工具以空 `profiles` 表示，并由 artifact 投影为 `source: hidden_handler`。这是旧 "hidden handler" 的终态替代口径：它们是 artifact-backed internal tools，不是“有 handler 但未暴露”的旧目录残留。

Book、Presentation 等 internal 工具是否公开到 MCP，是调整 tool-module profile 的产品决策，不是恢复旧 handler/type 结构。

### 3. MCP 入口统一

MCP 入口统一走：

```text
src/index.js 或 src/advanced/index.js
  -> src/core/mcpServer.js
  -> src/core/toolRouter.js
  -> src/tools/index.js registry
  -> tool-module handler
```

`src/index.js` 只传 `profile: 'classic'`；`src/advanced/index.js` 只传 `profile: 'advanced'`。两个入口不维护独立工具映射。

### 4. artifact 单向投影

CLI Node-backed 工具目录只读：

```text
src/core/indesign-tool-registry.json
```

artifact 由 registry 单向生成：

```powershell
node src/core/artifact.js --write
node src/core/artifact.js --check
```

Python CLI 可以叠加 CLI primitives 和 plugin runtime overlay，但不得为 Node-backed 工具重新推断 domain、schema、contract 或 hidden/internal 工具。

artifact 缺失、JSON 无效、`registry_hash` 不匹配或数量不一致都是硬错误。

### 5. 无 fallback 原则

终态架构不保留旧结构 fallback：

- 不恢复 `src/handlers/`
- 不恢复 `src/types/`
- 不恢复 `src/core/InDesignMCPServer.js`
- 不恢复 Python `domains.py` 推断
- 不恢复 Python hidden schema
- 不新增兼容 facade 或第二份 registry

出现 artifact 漂移时，正确处理是重新生成并校验 artifact，而不是绕过 artifact 让消费方猜测。

## 影响范围

- MCP server：只通过 `mcpServer.js` + `toolRouter.js` 调用 registry。
- CLI：Node-backed 工具目录只读 artifact；CLI primitives 和插件保持 overlay。
- 测试：`scripts/check_architecture.mjs`、`tests/architecture/registry.test.mjs`、`tests/index.js --required`、CLI pytest 是当前基线。
- 文档：README、Skill、MCP 文档和 LLM prompt 只能描述当前终态口径；旧架构资料只作为 `docs/legacy/` 或已取代计划追溯。

## 验证方式

```powershell
git diff --check
node src/core/artifact.js --check
node scripts/check_architecture.mjs
node tests/architecture/registry.test.mjs
node tests/index.js --required
python -m pytest agent-harness\cli_anything\indesign\tests -q
```

触及真实 InDesign 行为时，再运行对应 `tests/real-e2e/` 阶段或全量真实 E2E，并在结果中说明是否运行。

## 后续事项

语义契约计划必须等终态架构重构合并 `master` 后执行。语义字段应作为 tool-module 的可选字段共置，并通过同一个 artifact 单向投影给 CLI；不得建立按 tool name 索引的中心语义字典。
