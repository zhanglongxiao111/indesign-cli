# Agent 反馈闭环流程

## 决策

`indesign-cli` 的 Agent 反馈闭环采用"白名单遥测 + 明确 feedback 上报 + 确定性聚合 + 周期分析 Agent + 带承接测试的修复"流程。

## 长期约束

- 遥测字段白名单制；新增字段必须 review。
- 禁止记录参数值、脚本内容、客户文档内容、客户名称、完整文件路径和完整工作目录。
- 共享目录只写 `sessions/**/*.jsonl`、`state/*.json` 和 `reports/` 聚合结果。
- 分析输出只使用 `origin_key`、`cwd_hash`、工具 id、错误码、耗时、参数键名和 feedback code/note。

## 快通道

以下修复可走快通道：

- 错误信息文本。
- `tool schema` description 文案。
- `tool explain` 返回说明。
- `skills/indesign-cli/SKILL.md` 行为提示。
- `README.md` / `README.en.md` 示例和说明。

快通道只能改文案或提示，不得改变工具行为、schema 结构、handler 执行链路或插件协议。

## 摩擦簇

分析 Agent 为达到阈值的摩擦分配簇 ID：`FC-YYYYMMDD-NN`。

任何修复 PR 必须引用对应簇 ID。没有簇 ID 的零散修复可以合并，但不能算作反馈闭环的指标修复。

## 承接测试

没有承接测试的修复不算闭环。

| 摩擦类型 | 承接测试 |
| -------- | -------- |
| 错误信息类 | 单元测试断言 `error.code`、`error.hint` 或 message 更可操作 |
| 工具缺口类 | 新增工具 schema 契约测试；触及真实 InDesign 行为时补 E2E 场景 |
| schema 类 | `tool schema` / `tool explain` 契约测试覆盖参数、required、description |
| 文档类 | README / Skill grep 或示例命令测试，确保说明不回退 |
| 聚合逻辑类 | `scripts/feedback/fixtures/expected.json` golden 对比 |

## 发版验证

修复发布后，分析 Agent 必须按 `cli_version` 切片复核：

- 对应簇的错误频次是否下降。
- 重试率是否下降。
- escape hatch 前序失败是否下降。
- feedback code 是否减少。

指标未下降时，簇状态标记为 `reopened`，下一期周报继续追踪。
